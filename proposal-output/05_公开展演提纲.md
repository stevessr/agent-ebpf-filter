# 公开展演提纲

> 本文件用于项目投稿答辩、公开展演和路演汇报，配合 `01_重点课题版.md`、`02_短版.md`、`03_封面目录导师意见模板.md`、`04_深度调研报告.md` 一起使用。

## 一、30 秒开场

我们做的是一个面向高风险稀缺样本识别的轻量级可解释表格模型系统。它不仅要识别风险样本，还要保证正常命令尽量放行，因此我们把 **ALLOW 放行率、少数类召回、误伤率、训练耗时和推理速度** 一起纳入评价。

## 二、5 分钟讲述结构

1. **问题是什么**
   - 命令执行、工具调用、运维审批场景里，漏判高风险命令会出事，误伤正常命令也会影响可用性。
2. **我们怎么做**
   - 用公开安全知识库、已有缓存样本、合成样本和人工复核样本构建训练集。
   - 比较 Random Forest、Extra Trees、Logistic、SVM、Ridge、KNN、Naive Bayes、Boosting 等轻量模型。
3. **系统做成了什么**
   - 已有可运行原型，支持导入、去重、脱敏、导出、训练和结果浏览。
   - 已整理 PPTX 风格 HTML 展示页和研究报告。
4. **目前结果如何**
   - 当前累计 949 条样本，稳定复核后 `svm_long` 是最稳的综合候选。
   - 单次速度上，`random_forest` 可作为高吞吐对照。
5. **为什么值得做**
   - 它把“安全”和“可用”放在一起评估，更贴近真实工具链和命令放行场景。

## 四、评审常问问题的回答方向

- **为什么不用只看 accuracy？**  
  因为安全类场景里，误伤正常命令同样会影响系统可用性，所以必须同时看 ALLOW 放行率和误伤率。

- **为什么优先轻量模型？**  
  因为申报和落地都需要低成本、可解释、易部署，轻量模型更适合首期提交和展示。

- **为什么这个项目不是纯调参？**  
  因为我们同时做了数据闭环、脱敏清洗、公开知识库导入、误判回流和系统展示，模型只是整个闭环中的一环。

- **后续还能怎么扩展？**  
  可以继续加入更多公开数据、主动学习复核池、时间切分验证，以及更强的 foundation model 对照。
