# 金种子科研项目申报书（短版）

> 本稿面向“华南师范大学学生课外科研金种子课题”类项目；详细版已补充 2024—2026 年 tabular benchmark、类别不平衡学习、命令安全检测和表格基础模型调研。

**核心卖点：** 不只看准确率，还看正确样本放行率（ALLOW）、少数类召回、误伤率、训练耗时与推理速度。

## 一句话版本

本项目面向工具调用、命令执行、运维审批等高风险稀缺样本识别场景，围绕不平衡表格数据构建一套**轻量、可解释、可复用**的模型比较与数据闭环系统。我们不只看准确率，还同时看 **ALLOW 放行率、少数类召回、误伤率、训练耗时与推理速度**，并系统比较 Random Forest、Extra Trees、Logistic、SVM、Ridge、KNN、Nearest Centroid、Naive Bayes、Boosting、Ensemble 及其 balanced 变种，最终形成可落地的原型系统和可用于结题/竞赛的成果包。

## 深度调研后的定位

相关研究显示：PMLBmini 证明低样本下简单模型仍有竞争力；CLIMB 强调不平衡任务要比较采样、代价敏感和 ensemble；TabArena 说明评测需要长期维护和缓存复现；GTFOBins、LOLBAS、MITRE ATT&CK、ShellCore、LOLAL 等工作说明命令安全检测有可借鉴的知识库、数据源和主动学习路线。但多数同类工作更关注“发现恶意”，较少把**正确命令能否顺利放行**作为主指标。本项目的差异化就在于：把 ALLOW 放行率和误伤率作为核心业务指标，并通过误判回流持续更新数据集。

## 核心亮点

- **问题真**：高风险样本少，但漏判代价高；正确样本也不能误伤。
- **指标全**：准确率 + 放行率 + 少数类召回 + 误伤率 + 速度 + 稳定性。
- **模型广**：树模型、线性模型、近邻模型、概率模型、Boosting、集成模型一起比。
- **数据活**：已有缓存 + 公开 shell command 数据 + GTFOBins/LOLBAS/ATT&CK 知识源 + 合成样本 + 人工复核。
- **结果实**：最后形成原型系统、图表、报告与结题材料。

## 研究内容

1. 整理数据集并建立 ALLOW / ALERT / BLOCK 标签规范。
2. 扫描 30+ 基础模型及其变种。
3. 用多维指标筛选更稳的方案。
4. 加入阈值校准、主动学习与误判回流。
5. 做成可展示、可复现、可继续迭代的原型系统。

## 预期成果

- 原型系统
- 研究报告 / 深度调研报告
- 数据处理、训练、参数 sweep 与可视化脚本
- 竞赛/论文/软著基础材料
