# 深度调研报告：高风险稀缺样本识别、命令安全放行与表格模型评测

> 用途：作为 `01_重点课题版.md` 的调研支撑材料，可在申报书正文中摘取“研究现状”“同类项目分析”“创新点”和“可行性分析”。  
> 调研时间：2026-05-07。  
> 调研范围：tabular benchmark、类别不平衡学习、命令/安全检测、公开命令数据、living-off-the-land 知识库、表格基础模型与 LLM 辅助解释。

## 1. 执行摘要

本课题处在三个研究交叉点：

1. **表格数据机器学习**：传统树模型和线性模型仍然是强 baseline，深度表格模型和 foundation model 正在快速追赶。
2. **类别不平衡与高风险稀缺样本**：只看 accuracy 容易得到“看似很高但业务不可用”的结果，需要同时报告少数类召回、误伤率、放行率和稳定性。
3. **命令/安全放行检测**：已有研究多关注恶意命令检测、LOTL 攻击、低 FPR 和命令解释，但较少把“正确命令必须放行”作为核心指标。

因此，本项目最适合形成的差异化口径是：

> **不是再做一个只追求拦截率的安全检测器，而是做一个面向工具调用/命令执行场景的“安全可用放行决策系统”：同时优化高风险识别和正常命令 ALLOW 放行率。**

## 2. 表格数据 benchmark 进展

| 项目 / 论文 | 关键信息 | 适合借鉴的点 | 对本课题的限制 |
|---|---|---|---|
| PMLB / PMLBmini / TabMini | PMLBmini 包含 44 个 ≤500 样本二分类数据集；低样本下简单 logistic baseline 仍很强 | 小样本实验、简单模型 baseline、数据效率分析 | 通用数据集，不直接覆盖命令安全语义 |
| CLIMB / IMBENS | CLIMB 包含 73 个真实数据集和 29 个类别不平衡算法；强调 naive rebalancing 局限 | class-weight、sampling、balanced ensemble、效率指标 | 不直接讨论 ALLOW 放行率 |
| TALENT | 提供 20+ 深度 tabular 方法和统一工具箱 | 若后续引入深度表格模型，可对齐其接口和模型族 | 首期部署可能过重 |
| A Closer Look at Deep Tabular | 300+ 数据集系统评测；树模型仍强，预训练模型在部分任务追上或超过 | 支持“传统强 baseline + 前沿对照”的路线 | 论文关注通用预测，不是安全放行 |
| TabularBench | 200+ 模型、5 类关键场景、7 类鲁棒化机制 | 加入扰动鲁棒性和现实约束 | 更偏对抗鲁棒性，不聚焦命令文本 |
| TabReD | 真实生产/Kaggle 数据、时间切分、特征工程、漂移 | 补充分来源/时间切分，避免随机切分过乐观 | 数据形态是工业表格，不是命令安全 |
| TabArena | living benchmark，51 个数据集、16 种方法、2500 万次训练模型和预测缓存 | 保存 sweep 结果、预测缓存、leaderboard 式持续迭代 | 资源规模大，学生项目需做轻量化版本 |
| RelBench / RelBench v2 | 多表关系数据，v2 扩展至 11 个数据集、2200 万+ 行、29 张表 | 后续可把单条命令扩展为进程、会话、文件、网络关系图 | 首期不宜直接做复杂关系学习 |

### 小结

这些 benchmark 的共识是：

- tabular 任务不能只比较单个模型；
- 数据切分、参数调优、ensemble 和缓存协议会显著影响结论；
- 低样本/不平衡场景下简单模型仍值得保留；
- foundation model 正在提升上限，但部署成本和可解释性仍要权衡。

## 3. 命令安全检测与同类项目

| 来源 | 研究对象 | 主要贡献 | 本项目可吸收的做法 |
|---|---|---|---|
| MITRE ATT&CK T1059 / T1059.004 | 命令与脚本解释器、Unix Shell | 提供权威攻击技术分类和检测策略 | 把标签解释对齐到 ATT&CK 技术/战术 |
| GTFOBins | Unix-like 可滥用二进制 | 按 Shell、Command、Reverse shell、Upload、Download、SUID 等功能组织 | 构造风险模板、解释样本来源、生成边界样本 |
| LOLBAS | Windows Living-off-the-Land 二进制、脚本、库 | 提供 ATT&CK 映射和程序化访问思路 | 拓展跨平台知识源，丰富规则和合成数据 |
| ShellCore | IoT 恶意 shell 命令 | term/char 特征 + 传统/深度模型，检测精度高 | 证明命令文本特征对风险识别有效 |
| LOLAL | Living-off-the-Land 命令检测 | 主动学习，从少量标签迭代选择不确定样本 | 建立人工复核池：优先标注不确定和高影响样本 |
| Robust Synthetic LOTL Reverse Shells | reverse shell 检测 | 合成扩增 + 对抗训练 + 低 FPR 目标 | 学习“合成数据 + 低误伤约束”的实验设计 |
| UCI / Zenodo Shell Commands | 真实训练场景 shell 命令 | 21,459 条命令，含参数、时间戳、目录等元数据 | 补充正常命令分布，提高 ALLOW 放行能力 |
| DistilBERT Shell Session Anomaly | Unix shell 会话异常 | 会话级 transformer，无监督/监督结合 | 二期扩展到命令序列和会话上下文 |
| RACONTEUR | LLM shell 命令解释 | 专业知识增强，输出行为/目的并映射 ATT&CK | 用于解释与辅助标注，避免作为唯一分类器 |

### 关键发现

安全检测文献中常见目标是“发现攻击”和“降低 FPR”，但在真实工具调用系统里，**误伤正常命令同样会导致系统不可用**。因此，本项目需要把指标拆为两类：

- **安全性指标**：BLOCK/ALERT 召回、漏放行率、恶意样本 F1；
- **可用性指标**：ALLOW pass rate、false block rate、推理延迟。

这比单纯报告 accuracy 或 F1 更适合申报书中的“问题价值”和“创新点”表述。

## 4. 表格基础模型与 LLM 拓展

| 项目 | 当前进展 | 对本课题的价值 | 风险与边界 |
|---|---|---|---|
| TabPFN / TabPFN-2.6 | 官方仓库默认模型已到 TabPFN-2.6；强调少预处理、GPU 推荐、适合小中型表格 | 作为高阶对照或教师模型 | GPU/延迟/隐私/模型体积不一定适合首期部署 |
| TabPFN-2.5 | 扩大到更大数据和特征规模，并提出蒸馏到小 MLP/树集成 | 蒸馏思路适合“高性能教师 → 轻量学生” | 论文报告口径仍需在本地数据复测 |
| TabICL | 面向大数据 ICL，在 TALENT 200 个分类数据集上接近 TabPFNv2 且更快 | 若数据增长，可作为大表格对照 | 仍属于前沿模型，工程接入成本高 |
| Real-TabPFN | 用少量真实数据继续预训练可提升效果 | 说明合成数据不是终点，真实高质量数据很重要 | 预训练不适合作为金种子首期目标 |
| TabLib | 大规模表格语料基础设施 | 支撑“表格预训练正在形成”的背景论证 | 与本项目命令安全数据语义距离较远 |
| TabLLM | 表格序列化 + LLM few-shot 分类 | 可做小样本对照或标签解释 | LLM 推理成本、幻觉和隐私需控制 |
| RACONTEUR | LLM 解释 shell 命令并映射 ATT&CK | 辅助人工复核和报告解释 | 不替代实时轻量分类器 |

### 小结

Foundation model 可以作为“研究上限”和“二期拓展”，但申报书首期更稳妥的技术路线仍是：

1. 轻量模型建立强 baseline；
2. 多指标稳定评测选出可部署模型；
3. 用 LLM / foundation model 辅助解释、标注或蒸馏，而非强依赖。

## 5. 本项目的研究问题重述

基于调研，本项目可把研究问题明确为：

1. 在高风险样本稀缺、正常样本占多数的场景中，哪些轻量模型能同时保持高准确率和高 ALLOW 放行率？
2. 在加入合成高风险样本、公开 shell command 数据和人工复核样本后，模型是否会提高召回但牺牲放行率？如何平衡？
3. balanced / class-weight / ensemble / threshold calibration 对少数类召回、误伤率和推理速度分别有什么影响？
4. 是否存在“稍低最高准确率但更高稳定性、更低误伤、更快推理”的最优工程方案？
5. 如何把 GTFOBins、LOLBAS、ATT&CK 等知识库转化为可解释标签、合成样本和模型错误分析，而不是停留在规则堆砌？

## 6. 建议的实验设计

### 6.1 数据切分

- **分层随机切分**：保证 ALLOW / ALERT / BLOCK 比例稳定；
- **来源切分**：缓存数据、公开数据、合成数据分别保留来源字段；
- **时间/版本切分**：如果数据有导入时间或命令生成版本，保留未来验证集；
- **难例集**：将历史误判和人工不确定样本作为固定 stress test。

### 6.2 模型 sweep

| 模型类 | 参数维度 |
|---|---|
| Random Forest / Extra Trees | `n_estimators`, `max_depth`, `min_samples_leaf`, `class_weight` |
| Logistic / Ridge | 正则项、C/alpha、class_weight、阈值 |
| Linear SVM / Perceptron / Passive Aggressive | 学习率、迭代次数、正则、class_weight |
| KNN / Nearest Centroid | k、距离、权重、标准化方式 |
| Naive Bayes | smoothing、特征类型 |
| AdaBoost / Ensemble | estimator 数、base learner、投票权重 |
| TabPFN / TabICL 对照 | 默认模型、少量数据子集、蒸馏可行性 |

### 6.3 指标报告格式

每个模型至少报告：

- accuracy 均值 ± 标准差；
- ALLOW pass rate 均值 ± 标准差；
- false block rate；
- BLOCK/ALERT recall；
- macro-F1 / balanced accuracy / MCC；
- 训练耗时；
- 推理吞吐；
- 100 次重复成功率；
- 最差 5% 分位点表现。

### 6.4 错误分析

将错误分为：

1. **误阻断**：应 ALLOW 却被 BLOCK / ALERT；
2. **漏放行**：应 BLOCK / ALERT 却被 ALLOW；
3. **标签不一致**：同类命令上下文相似但标签冲突；
4. **合成数据偏差**：模型学习了模板痕迹而不是风险语义；
5. **上下文缺失**：单条命令无法判断，需要工作目录、父进程、用户、环境变量或会话信息。

## 7. 申报书可用结论段落

可直接贴入“国内外研究现状”或“立项依据”：

> 近年表格数据研究呈现出三个趋势：一是 benchmark 从静态数据集走向 living benchmark，强调复现、缓存预测和长期维护；二是类别不平衡任务从单纯重采样走向数据质量、代价敏感、ensemble 和多指标评估；三是 TabPFN、TabICL 等表格基础模型正在提高小样本和中等规模表格任务的性能上限。与此同时，在命令安全检测领域，MITRE ATT&CK、GTFOBins、LOLBAS、ShellCore、LOLAL、RACONTEUR 等工作分别提供了攻击技术分类、可滥用命令知识库、命令文本检测、主动学习和 LLM 解释等支撑。但现有工作大多更关注“识别恶意”或“降低误报”，较少把正确命令的业务放行率作为核心评价指标。因此，本项目将 ALLOW 放行率、误伤率、少数类召回和推理速度一起纳入主指标，构建面向真实工具调用场景的轻量可解释表格模型与数据闭环系统。
