Research scope

更大的搜索空间，更严格的评价口径

本轮 sweep 统计 1346 条单次训练结果，重点展示扩大的搜索空间和参数组合总量，同时在演示页中加入 31 个模型族的代表性变体。

研究如何被扩大

  • 从单点验证扩展为 全模型横向 sweep，覆盖树模型、线性模型、KNN、Ridge、AdaBoost、Naive Bayes 与 Ensemble，并拆出更多变体家族。
  • 树模型参数改为更宽的 numTrees × maxDepth 网格；线性模型扩大 learning rate × iterations 的搜索范围。
  • 演示页额外抽取每个 family 的 top 4 配置，形成 106 个代表性模型变体 的可视化画廊。
  • 对每个模型族选出的最优点，再做 稳定性重复 观察均值、标准差和成功率。
  • 最终判断不再只看单次准确率，而是看 稳定均值、ALLOW 放行率、方差、推理速度、耗时 和数据集可编辑性。

03
Literature

参考最新论文，继续扩展基础模型

这一页把近期 tabular benchmark 的共识，翻译成这次基础模型扩展与参数搜索的工程选择。

最新论文给了什么信号

  • CLIMB (2025)：类不平衡 tabular 任务里，单纯重采样不一定有效，ensemble 往往更稳；评价指标不能只看准确率。
  • PMLBmini (2024)：在低数据区间里，简单逻辑回归仍然是强基线，AutoML / deep learning 并不总能稳定压过它。
  • TabArena (2025)：living benchmark 的核心不是堆榜，而是持续更新数据集、模型与评测协议，并把 tuning + ensembling 看成真实上限的一部分。
  • 综合基准 (2024)：传统树模型依旧很强，但在充分调参与合并后，深度模型在部分 tabular 任务上也能接近甚至超过它们。
  • 对本仓库的直接启发：我把线性、树、近邻、朴素贝叶斯的“基础模型 + 变种”一起拉进来，并同时看准确率、ALLOW 放行率、推理速度与训练耗时。

论文 → 工程映射

论文关键结论本仓库里的对应动作
CLIMB 2025不平衡数据上，单纯重采样未必提升；ensemble 更稳，指标不能只看 accuracy。增加 balanced 变体，并把 ALLOW 放行率纳入主表。
PMLBmini 2024小样本 tabular 里，逻辑回归仍然常常是强基线。保留并扩展 logistic / ridge / nearest centroid 这类轻量基线。
TabArena 2025living benchmark 要持续更新数据、模型和协议，而不是静态堆分数。把 sweep、repeat、图表、HTML 演示页做成可复用流程。
综合基准 2024树模型仍强，但调参 + 合并后，深度模型在部分 tabular 任务可竞争。对随机森林 / Extra Trees / ensemble 做更宽参数扫描。
注：最后一列是基于论文结论做出的工程映射（推断），不是论文原文。
