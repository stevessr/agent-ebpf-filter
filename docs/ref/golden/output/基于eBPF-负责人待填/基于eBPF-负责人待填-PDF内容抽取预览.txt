本项目面向生成式人工智能 Agent 在本地开发环境、实验室服务器和教学机房中
自动执行命令、读写文件、派生子进程所带来的运行时安全风险，拟研发一套以
BPF LSM 策略执行、内核态路径匹配和进程树追踪为核心的 AI Agent 系统调用过
滤与监控平台。平台不只停留在事件展示，而是把安全判断前移到内核执行路
径 ： 通 过 eBPF tracepoint/LSM hook 捕 获 execve 、 openat/openat2 、 connect 、
mkdirat、unlinkat、ioctl、bind、sendto、recvfrom 等关键行为，在 Go 后端维护策
略、身份和审计链路，在前端提供必要的配置与教学展示。

主要研究内容包括：
1. BPF LSM 策略执行：围绕 file_open、inode_permission、bprm_check_security、
socket_connect 等 LSM 挂载点设计策略执行链，将 ALLOW、BLOCK、ALERT、
REWRITE/降级处理等决策与系统调用上下文关联，研究在内核态完成快速拒

主要研究
绝、在用户态完成复杂裁决的协同机制。
内容

2. 内核态路径匹配：针对 AI Agent 访问项目源码、密钥、数据集和系统目录的风
险，研究 dentry/mount 路径解析、BPF map 编码、固定长度路径精确匹配、目录
前缀匹配和路径标签缓存机制，减少每次文件访问回退到用户态判断的开销。
3. 进程树追踪：以注册的 Agent 主进程为根，跟踪 fork、clone、exec 形成的父子
链路，维护 task lineage 与会话上下文，使策略能继承到 shell、包管理器、编译
器、脚本解释器等子进程，避免只监控顶层 PID 造成绕过。
4. 策略控制与审计闭环：在 wrapper、AI CLI hook 和 eBPF 事件之间建立统一事件
模型，记录策略命中、阻断原因、路径标签、进程树节点和工具调用标识，支持
复现实验和安全审计。
5. 配套可视化与教学转化：在完成上述内核态安全能力后，利用仪表盘、执行图
和实验文档展示策略执行过程，服务课程实践、竞赛路演和成果推广。

本项目的目的，是在 AI Agent 快速进入科研、开发和教学场景的背景下，验证一
条“内核态可执行策略 + 进程谱系追踪 + 路径级访问控制”的本地安全技术路
线。现有 AI 编程助手和自动化代理能够调用 shell、安装依赖、读取工程文件和访
问网络，但其实际行为往往由多级子进程完成，单纯依赖应用层日志或前端展示
难以阻止越权文件访问、危险命令执行和异常网络外联。

基本思路为：
1. 以 BPF LSM 作为关键执行点，在文件打开、权限检查、程序装载和网络连接等

项目研究
路径上完成可验证、低开销的策略判断。

基本思路 2. 以内核态路径匹配作为核心能力，设计路径规范化、前缀/精确匹配、标签缓存
和 map 热更新机制，使敏感目录保护能够在高频文件访问场景下稳定运行。
3. 以进程树追踪作为归因基础，将 Agent 根进程、shell、脚本解释器、编译器、
包管理器和工具进程串联为可继承的安全上下文。
4. 以后端策略引擎和 wrapper/hook 作为补充，对复杂规则、提示链路、工具调用
意图和人工确认流程进行协同处理。
5. 以可视化界面、实验教程和文档作为配套成果，展示内核策略命中、路径访问
链路和进程树传播过程，支撑教学与成果转化。

国内外关于 eBPF、BPF LSM 和运行时安全的研究与实践发展迅速。 Cilium、
Falco、Tetragon 等项目证明了 eBPF 在低开销观测、进程/文件/网络事件关联和云
原生安全中的工程价值；Linux BPF LSM 则进一步提供了在内核安全决策点执行
策略的能力，使 eBPF 从“看见行为”走向“约束行为”。另一方面，生成式 AI
Agent 正在从对话辅助走向可执行任务代理，其风险常通过子进程、脚本解释器、
包管理器和临时文件访问链路体现。

当前同类研究主要集中在三类方向：一是传统主机入侵检测和 EDR，强调通用进

当前国内外
同类课题 程、文件、网络事件，但对 Agent 根进程到子进程的任务链归因不足；二是云原
研究情况 生 eBPF 安全产品，偏重容器和生产集群，策略模型较重，在高校本地开发环境部
署门槛较高；三是 AI 安全与提示注入研究，更多关注模型输入输出，对系统调用
层面的强制访问控制和路径策略执行关注不足。

本课题的差异在于：不把可视化作为核心创新，而是以 BPF LSM 策略执行、内核
态路径匹配和进程树追踪为三条主线，构建面向 AI Agent 的本地运行时安全原
型；可视化、实验教程和竞赛展示作为配套工作，用于解释和验证底层安全机
制。

拟解决的关键问题包括：
1. Agent 子进程绕过与归因困难问题：通过 PID 注册、fork/clone/exec 追踪和
lineage 继承，将 Agent 根进程与其派生的 shell、解释器、编译器、包管理器等纳
