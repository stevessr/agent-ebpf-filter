二、选题背景与意义

生成式 AI Agent 已经从“回答问题”发展到“执行任务”：它们会自动调用 shell、启动脚本解释器、安装依赖、读取项目文件、访问网络服务，并通过多级子进程完成真实操作。对于高校实验室、教学机房和本地开发环境而言，风险不只在模型输出，而在 Agent 运行时是否访问了敏感路径、是否派生了危险子进程、是否越权连接网络。传统日志审计和前端可视化只能事后说明“发生了什么”，难以及时回答“是否应该允许这次内核操作”。

本项目拟基于现有 agent-eBPF-filter 工程，重点研发三个硬技术能力：

1. **BPF LSM 策略执行**：在 file_open、inode_permission、bprm_check_security、socket_connect 等 Linux Security Module 挂载点执行策略，使平台从系统调用观测进一步走向内核态强制访问控制。
2. **内核态路径匹配**：围绕 AI Agent 对源码、密钥、数据集、系统目录的访问控制，研究路径规范化、固定长度精确匹配、目录前缀匹配、BPF map 编码、路径标签缓存与热更新，减少高频 openat/openat2 场景下的用户态往返。
3. **进程树追踪**：以注册的 Agent 主进程为根，跟踪 fork、clone、exec 形成的父子链路，将 shell、脚本解释器、编译器、包管理器和工具进程纳入同一安全上下文，避免只监控顶层 PID 造成绕过。

可视化、执行图、仪表盘和教学实验材料作为配套工作，用于解释策略命中、路径访问链路和进程树传播过程，而不是本项目的主要创新点。

三、项目目标

- 研发一套面向 AI Agent 的运行时安全平台，使策略能够在 BPF LSM 执行点完成快速放行、阻断、告警或降级处理。
- 实现可热更新的内核态路径匹配能力，支持敏感路径精确匹配、目录前缀策略、路径标签和策略审计。
- 建立进程树追踪与策略继承机制，使 Agent 派生的子进程自动继承安全上下文和路径/网络/命令策略。
- 构建性能评测体系，量化 LSM 阻断延迟、路径匹配开销、进程树维护成本和不同内核版本兼容性。
- 形成论文、软著或专利材料；配套完成用户手册、实验教程和展示视频，用于教学转化和竞赛申报。

四、研究内容与技术路线

4.1 BPF LSM 策略执行

在现有 tracepoint 观测基础上，引入 BPF LSM hook 作为安全决策入口。重点分析 file_open、inode_permission、bprm_check_security、socket_connect 等挂载点可获得的内核上下文，设计策略执行链：内核态完成低开销、可验证的快速判断；用户态后端负责复杂规则、人工确认、wrapper/hook 语义信息和审计归档。策略结果包括 ALLOW、BLOCK、ALERT、REWRITE/降级处理等，并通过 ring buffer 或事件通道回传命中原因。

4.2 内核态路径匹配

现有 tracked_paths 采用固定长度精确匹配，适合低成本路径标签，但无法充分覆盖敏感目录和多层项目结构。本课题将研究路径规范化、dentry/mount 解析、路径哈希、分段匹配、目录前缀匹配、BPF trie/LPM 或多级 hash map 等结构，评估不同方案在 eBPF verifier 约束、内存占用和 openat 高频调用下的性能。对于复杂正则或语义规则，保留在 Go 后端处理，避免在内核态引入不可控开销。

4.3 进程树追踪与策略继承

通过 fork、clone、exec 事件维护 Agent 进程树，记录父子 PID、comm、执行路径、会话标识和工具调用标识。策略从 Agent 根进程向子 shell、Python/Node 解释器、编译器、包管理器等派生进程继承，并在进程退出时清理状态。该机制用于解决 AI Agent 通过间接命令执行绕过监控的问题，也是后续审计和执行图展示的基础。

4.4 策略控制、审计与兼容性

后端负责维护策略版本、BPF map 热更新、运行时令牌、wrapper 控制和 hook 回调。平台需要处理不同内核版本是否支持特定 LSM hook/tracepoint 的差异：具备能力时优先内核态执行，缺失能力时降级为 tracepoint 观测加用户态策略，并在审计日志中标明降级原因。

4.5 配套可视化与教学转化

前端仅作为策略调试、教学演示和实验数据采集入口：仪表盘展示策略命中、路径标签、阻断原因；执行图展示 Agent 根进程到子进程的传播链；教程解释 BPF LSM、路径匹配和进程树追踪的实验步骤。教学推广不替代底层技术研发。

五、创新点

1. 将 BPF LSM 用于 AI Agent 本地运行时策略执行，使平台从“监控系统调用”推进到“在内核安全决策点约束行为”。
2. 面向 Agent 文件访问特点设计内核态路径匹配与路径标签缓存机制，在性能与策略表达力之间取得平衡。
3. 建立 Agent 进程树追踪和策略继承模型，解决子进程、解释器、包管理器绕过顶层 PID 监控的问题。
4. 将 wrapper、AI CLI hook、eBPF 事件和后端策略统一为可审计闭环，能够回答“哪个 Agent 的哪条任务链触发了哪条策略”。
5. 可视化和教学材料围绕底层安全机制展开，便于成果展示、课程实验和竞赛转化。

九、可行性分析与风险应对

技术可行性方面，eBPF、BPF LSM、BPF map 和 ring buffer 已经在 Linux 安全与云原生观测中广泛使用，本项目基于现有 agent-eBPF-filter 工程开展二次研发，可降低从零构建平台的风险。团队可从 tracepoint 观测和最小 LSM hook 原型切入，再逐步完善路径匹配与进程树继承。

主要风险包括：一是不同内核版本对 BPF LSM 支持不一致，项目将采用 hook 可用性探测、缺失能力降级和用户态补偿策略；二是路径匹配在高频文件访问中可能引入开销，项目将通过精确匹配、前缀匹配、缓存和基准测试逐步优化；三是进程树状态可能因短生命周期进程频繁创建而膨胀，项目将设计退出清理、TTL 和容量限制机制。

十、结语

本项目紧密围绕 BPF LSM 策略执行、内核态路径匹配和进程树追踪三个底层技术问题，面向 AI Agent 本地运行时安全提出可实现、可评测、可转化的研发方案。可视化和教学推广作为配套环节，将帮助学生理解和展示内核安全机制，但项目核心仍是系统级策略执行能力的突破。恳请学校评审老师批准立项。
